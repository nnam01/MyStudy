Linux 공부

네트워크 관련 명령어

  • ping: ping [호스트] – 네트워크 연결을 테스트합니다.
  • curl: curl [URL] – URL로부터 데이터를 가져옵니다. + 웹페이지 소스 확인, 파일 다운로드, POST요청 등 할 수 있음.
  • wget: wget [URL] – 웹에서 파일을 다운로드합니다.
  • nslookup: nslookup [도메인] – 도메인의 IP 주소 정보를 확인합니다.
  • netstat: netstat -tuln – 네트워크 상태와 포트 정보를 확인합니다.
  • ssh: ssh [사용자]@[호스트] – 원격 서버에 SSH로 접속합니다.
  • scp: scp [파일] [사용자]@[호스트]:[경로] – 원격 서버로 파일을 복사합니다.

파일 및 디렉토리 관리

  • cd: cd [디렉토리] – 현재 작업 디렉토리를 변경합니다.
  • ls: ls – 현재 디렉토리의 파일 및 디렉토리 목록을 표시합니다.
  • mkdir: mkdir [디렉토리명] – 새 디렉토리를 생성합니다.
  • rm: rm [파일] – 파일을 삭제합니다.
  • rm -r: rm -r [디렉토리] – 디렉토리와 그 안의 모든 파일을 삭제합니다.
  • cp: cp [원본] [대상] – 파일을 복사합니다.
  • mv: mv [원본] [대상] – 파일이나 디렉토리를 이동하거나 이름을 변경합니다.

프로세스 관리

  • ps: ps aux – 현재 실행 중인 프로세스 목록을 확인합니다.
  • kill: kill [PID] – 지정한 PID의 프로세스를 종료합니다.

파일 검색 및 수정

  • grep: grep [패턴] [파일] – 파일에서 패턴을 검색합니다.
  • find: find [경로] -name [파일명] – 지정한 경로에서 파일을 검색합니다.
  • sed: sed 's/old/new/g' [파일] – 파일에서 문자열을 치환하거나 수정합니다.
  • cat: cat [파일] – 파일의 내용을 출력합니다.
  • echo: echo "문자열" – 문자열을 출력합니다.
  • head: head -n [숫자] [파일] – 파일의 처음 몇 줄을 출력합니다.
  • tail: tail -n [숫자] [파일] – 파일의 마지막 몇 줄을 출력합니다.
  • diff: diff [파일1] [파일2] – 두 파일의 차이를 비교합니다.
  • man: man [명령어] – 해당 명령어의 매뉴얼(도움말)을 보여줍니다.

파일 권한 및 소유권 관리

  • chmod: chmod [권한] [파일] – 파일의 권한을 변경합니다.
  • chown: chown [사용자]:[그룹] [파일] – 파일의 소유자와 그룹을 변경합니다.

파일 압축 및 디스크 관리

  • tar: tar -cvf [아카이브명.tar] [디렉토리] – 디렉토리를 tar 아카이브로 압축합니다.
  • tar: tar -xvf [아카이브명.tar] – tar 아카이브를 압축 해제합니다.
  • zip: zip [아카이브명.zip] [파일] – 파일을 zip 아카이브로 압축합니다.
  • unzip: unzip [아카이브명.zip] – zip 아카이브를 압축 해제합니다.
  • df: df -h – 파일 시스템의 디스크 사용량을 표시합니다.
  • du: du -sh [디렉토리] – 디렉토리의 크기를 요약하여 표시합니다.

시스템 상태 확인

  • stat: stat [파일] – 파일의 상태 정보를 표시합니다.
  • file: file [파일] – 파일의 타입을 확인합니다.
  • which: which [명령어] – 명령어의 경로를 확인합니다.

시스템 관리

  • systemctl: 시스템 및 서비스 관리 명령어입니다.
  • 서비스: 시스템에서 실행되는 백그라운드 프로세스입니다. 예를 들어, 웹 서버, 데이터베이스 서버 등이 있습니다. jar파일 등을 중지하고 재실행 할 수 있음.
  • systemctl start [서비스]: 서비스를 시작합니다.
  • systemctl stop [서비스]: 서비스를 중지합니다.
  • systemctl restart [서비스]: 서비스를 재시작합니다.
  • systemctl status [서비스]: 서비스의 상태를 확인합니다.
  • systemctl enable [서비스]: 부팅 시 자동으로 서비스를 시작하도록 설정합니다.
  • systemctl disable [서비스]: 부팅 시 자동으로 서비스를 시작하지 않도록 설정합니다.
    • //systemctl daemon
  • systemctl daemon-reload: 서비스 파일을 변경한 후, 시스템 데몬을 다시 로드합니다.

시스템 모니터링

  • top: 시스템의 현재 프로세스와 리소스 사용량을 실시간으로 모니터링합니다.
  • htop: htop – top 명령어의 대안으로, 더 직관적인 인터페이스를 제공합니다.
  • free: free -h – 시스템의 메모리 사용량을 확인합니다.
