Git 공부

Git은 분산 버전 관리 시스템으로, 소스 코드의 변경 이력을 관리하는 데 사용됨.

생활코딩의 "지옥에서온 GIT"을 바탕으로 학습하였음.

기본 명령어

  • git init: 새로운 Git 저장소를 초기화합니다.
  • git clone [url]: 원격 저장소를 복제합니다.
  • git add [파일]: 변경된 파일을 스테이징 영역에 추가합니다.
  • git commit -m "메시지": 스테이징 영역의 변경 사항을 커밋합니다. → 버전 만들기
  • git commit -am "메시지": 변경된 파일을 스테이징하고 커밋합니다. (새로 추가된 파일은 제외)
  • git commit --amend: 마지막 커밋을 수정합니다.
  • git push: 로컬 커밋을 원격 저장소에 푸시합니다.
  • git pull: 원격 저장소의 변경 사항을 가져옵니다.
  • git fetch: 원격 저장소의 변경 사항을 가져오지만, 자동으로 병합하지 않습니다.
  • git status: 현재 작업 디렉토리의 상태를 확인합니다.

브랜치 관리

  • git branch: 현재 브랜치 목록을 확인합니다.
  • git checkout [브랜치명]: 다른 브랜치로 전환합니다.
  • git branch -d [브랜치명]: 지정한 브랜치를 삭제합니다.
  • git branch -m [새 브랜치명]: 현재 브랜치의 이름을 변경합니다.
  • git checkout -b [새 브랜치명]: 새 브랜치를 생성하고 전환합니다.
  • git switch [브랜치명]: 다른 브랜치로 전환합니다.
  • git switch -c [새 브랜치명]: 새 브랜치를 생성하고 전환합니다.
참고: git switch는 브랜치 전환과 생성에 특화된 명령어입니다.
git checkout은 브랜치 전환 외에도 파일 복원 등 다양한 용도로 사용됩니다.

병합 및 충돌 해결

  • git merge [브랜치 A]: 지정한 브랜치 A를 현재 브랜치에 병합합니다.
  • git merge --abort: 병합 중 충돌이 발생했을 때, 병합을 중단하고 이전 상태로 되돌립니다.
  • git cherry-pick [커밋ID]: 특정 커밋을 현재 브랜치에 적용합니다.
Fast-forward 병합: 브랜치가 직선적으로 이어져 있을 때, 병합 시 새로운 커밋 없이 포인터만 이동
Recursive 병합: 브랜치가 분기되었다가 다시 합쳐질 때, 새로운 커밋을 생성하여 병합

충돌 해결 절차:

  1. 충돌이 발생한 파일을 열어 충돌 부분을 수정
  2. git add [파일]로 변경 사항 스테이징
  3. git commit으로 병합 커밋 생성

3-way merge: base(공통 조상), me(현재 브랜치), other(병합 대상)를 비교하여 병합

git rebase [브랜치명]: 현재 브랜치의 변경 사항을 지정 브랜치 위로 재배치 → 히스토리를 깔끔하게 유지 가능

커밋 로그 및 비교

  • git log: 커밋 이력을 확인
  • git log -p: 각 커밋의 변경 내용을 확인
  • git log --oneline: 커밋을 한 줄로 요약하여 표시
  • git log --branches --decorate --graph: 브랜치와 커밋을 시각적으로 표시
  • git reflog: 로컬 저장소의 참조 로그 확인
  • git show [커밋ID]: 특정 커밋의 상세 정보 확인
  • git diff: 변경 내용 비교
  • git diff [커밋ID1] [커밋ID2]: 두 커밋 간 차이 비교

되돌리기 및 임시저장

  • git reset [커밋ID]: 지정한 커밋으로 되돌리기 (로컬에서만 사용 권장)
  • git reset --hard [커밋ID]: 지정 커밋 이후의 변경 사항 완전 삭제
  • git reset --soft [커밋ID]: 커밋만 취소, 변경 사항은 유지
  • git revert [커밋ID]: 변경을 취소하는 새로운 커밋 생성
  • git stash: 현재 작업 중인 변경 사항을 임시로 저장합니다.
  • git stash pop: 마지막에 저장한 스태시를 복원합니다.
